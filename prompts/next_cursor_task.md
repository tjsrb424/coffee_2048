## 왜 이 작업을 먼저 하는가

현재 프로젝트는 이미:

- 2048 퍼즐 코어
- 퍼즐 보상 연결
- 카페 운영 기본 루프
- 손님 최소 메타
- 성장 구조 4차 전달문 범위
- 메타 저장 persistence 회귀
- 손님 판매 write path 회귀(애정도/스토리/단골 전환)

까지 진행된 상태다.

이 시점에서는 새 기능을 바로 얹는 것보다,  
**Codex가 만든 실제 구조를 Cursor가 정확히 이해하고 정리하는 것**이 우선이다.

문서와 실제 코드가 어긋난 상태에서 다음 기능(핵심손님 확장, 특별 원두, 주문 시스템 등)을 얹으면
중복 구현 / 상태 충돌 / 저장 구조 꼬임이 생기기 쉽다.

## 구체 작업 지시

### 1. 현재 코드 구조 분석
아래를 먼저 정리해라.

- 레벨/미션 상태가 어디에 저장되는지
- 레시피 해금/구매 상태가 어디에 저장되는지
- 재료 인벤토리가 어디에 저장되는지
- 도감 상태가 어디에 저장되는지
- 스킨 구매/장착 상태가 어디에 저장되는지
- 저장/불러오기와 버전 키가 어디 있는지

### 2. 문서-코드 차이점 정리
다음을 구분해서 정리해라.

- **실제로 완료된 것**
- **미완료**
- **placeholder / TODO**
- **버그 가능성**
- **중복 구조 위험**

### 3. 작은 범위의 안정화 수정
문서와 실제 코드가 다르거나, 다음 단계 진행을 막는 이슈가 있으면 아래만 수정해라.

- 손님 판매 write path에서만 드러나는 저장 누락
- 잘못된 초기값
- `isRegular` / 스토리 / 선호 보너스 / daily quota 중 하나의 실제 판매 경로 불일치
- 디버그 import 기반 안정화가 필요하면 비영속 입력만 최소 추가
- 구조 리라이트 없이 Playwright에서 고정 가능한 범위만 다룰 것

### 4. 문서 동기화
`docs/codex_4th_pass_handoff.md`를 실제 코드 기준으로 업데이트해라.

반드시 반영:
- 실제 변경 파일 목록
- 실제 저장 구조
- 실제 구현 완료 범위
- 남은 리스크
- 다음 작업 추천 3개
