# 08. DEVELOPMENT ROADMAP

## 전체 전략

이 프로젝트는 처음부터 완성형으로 가면 실패 확률이 높다.
반드시 **손맛 MVP -> 연결 MVP -> 메타 확장** 순으로 가야 한다.

---

## 개발 단계 요약

### Phase 0. 사전 설계
- 비전 문서 작성
- 코어 루프 확정
- 타일 체인/메뉴 체인 확정
- 폴더 구조 생성
- Cursor 작업 규칙 문서화

### Phase 1. 2048 촉감 MVP
- 보드 생성
- 스와이프
- 이동/합체
- 점수
- 게임오버
- 타일 애니메이션
- 사운드/진동 추상화

### Phase 2. 퍼즐 보상 연결
- 세션 종료 결과
- 코인/원두 보상
- 세션 목표 1종
- 결과 모달
- 로비 자원 반영

### Phase 3. 카페 로비 MVP
- 기본 로비 화면
- 로스터
- 메뉴 3종 제작
- 재고 표시
- 자동 판매
- 코인 증가 연출

### Phase 4. 성장 구조
- 업그레이드
- 매장 레벨
- 메뉴 해금
- 저장/불러오기
- 오프라인 보상 간단 버전

### Phase 5. 생활감 강화
- 고객 등장/이동
- 배경 연출
- 꾸미기 1차
- 화면 전환 품질 향상

### Phase 6. 확장
- 주문 시스템
- 특수 타일 확장
- 이벤트
- 시즌
- 꾸미기 확장
- BM 연결

---

## 각 Phase 상세

## Phase 0. 사전 설계

### 목표
개발 전에 Cursor가 흔들리지 않도록 기준 문서를 만든다.

### 할 일
- docs 폴더 생성
- prompts 폴더 생성
- 게임 필러 정리
- 기술 스택 확정
- 디자인 키워드 정리
- 작업 금지사항 명시

### 완료 기준
- Cursor가 참고할 문서가 존재한다.
- 이후 기능 작업을 작은 단위로 자를 수 있다.

---

## Phase 1. 2048 촉감 MVP

### 목표
“일단 만졌을 때 기분 좋다”를 증명한다.

### 구현 목록
- 4x4 보드
- 새 타일 생성
- 상하좌우 이동
- 합체 로직
- 점수 계산
- 게임오버 판정
- 애니메이션용 좌표 데이터
- 키보드 + 터치 스와이프 입력

### UX 필수 항목
- 타일 이동 부드러움
- 합체 압축감
- 생성 팝
- 점수 상승 반응

### 완료 기준
- 단순한 2048이 아니라 “손맛 있는 2048”이 된다.

---

## Phase 2. 퍼즐 보상 연결

### 목표
퍼즐이 메타에 의미를 갖기 시작한다.

### 구현 목록
- 세션 결과 계산
- 보상 산출
- 골든빈 같은 특수 타일 1종
- 세션 목표 1종
- 결과 모달
- 로비 자원과 연결

### 완료 기준
- 퍼즐 플레이 이유가 생긴다.
- 점수 외의 동기가 생긴다.

---

## Phase 3. 카페 로비 MVP

### 목표
퍼즐 결과가 실제 매장 운영으로 이어진다.

### 구현 목록
- 로비 화면
- 자원 HUD
- 로스터 패널
- 메뉴 생산 패널
- 진열/재고 숫자
- 자동 판매 로직
- 코인 획득 피드백

### 완료 기준
- 퍼즐 -> 자원 -> 메뉴 -> 판매 -> 코인 루프가 된다.

---

## Phase 4. 성장 구조

### 목표
다시 돌아올 이유를 만든다.

### 구현 목록
- 업그레이드 패널
- 매장 레벨
- 메뉴 해금
- 고객 수 증가
- 저장/불러오기
- 기본 설정(사운드/진동)

### 완료 기준
- 플레이 누적 가치가 생긴다.

---

## Phase 5. 생활감 강화

### 목표
이 게임만의 감성적 경쟁력을 만든다.

### 구현 목록
- 고객 캐릭터
- 로비 배경 생동감
- 작은 연출들
- 로비/퍼즐 전환 정교화
- 수치 변화 고급화
- 모션 통일

### 완료 기준
- “게임이 예쁘다”가 아니라 “살아 있다”는 느낌이 든다.

---

## Phase 6. 확장

### 목표
리텐션과 콘텐츠 수명을 늘린다.

### 구현 목록
- 주문 시스템
- 데일리/주간 미션
- 특수 타일 다변화
- 이벤트 테마
- 꾸미기 확장
- 시즌 시스템

---

## 가장 먼저 만들어야 할 실제 작업 순서

1. 프로젝트 생성
2. Tailwind / Framer Motion / Zustand 세팅
3. 공통 디자인 토큰 만들기
4. 2048 엔진 함수 만들기
5. 보드 UI 만들기
6. 입력 처리
7. 합체 애니메이션
8. 결과 보상 연결
9. 로비 화면 만들기
10. 생산/판매 연결
11. 업그레이드
12. 저장

---

## 2주 단위 예시 스프린트

### Sprint 1
- 프로젝트 셋업
- 공통 UI 베이스
- 2048 엔진

### Sprint 2
- 2048 애니메이션
- 사운드
- 세션 목표/결과

### Sprint 3
- 로비 화면
- 자원 연동
- 메뉴 생산

### Sprint 4
- 판매/업그레이드
- 저장
- 최소 감성 polish

---

## 검수 체크포인트

각 Phase 종료 때 반드시 아래를 본다.

### 기능 체크
- 의도한 루프가 성립하는가
- 버그 없이 플레이 가능한가

### 감성 체크
- 반응이 밋밋하지 않은가
- 너무 복잡해지지 않았는가
- 힐링 감성이 깨지지 않았는가

### 구조 체크
- 코드가 분리되어 있는가
- Cursor가 다음 작업을 이어받기 쉬운가

---

## 주의사항

- 처음부터 백엔드 붙이지 않는다.
- 처음부터 BM 넣지 않는다.
- 처음부터 고객/주문/이벤트를 다 넣지 않는다.
- “예쁜 쓰레기”가 되지 않게 퍼즐 손맛부터 만든다.

---

## 결론

이 프로젝트의 개발 순서는 항상 아래로 기억하면 된다.

**손맛 -> 연결 -> 운영 -> 성장 -> 생활감 -> 확장**
