# 05. UX / MOTION DIRECTION

## 이 프로젝트의 UX 핵심

이 게임은 시스템보다 먼저 **손맛과 분위기**로 기억되어야 한다.
즉, 유저는 아래를 먼저 느껴야 한다.

- 터치가 기분 좋다
- 화면 전환이 부드럽다
- 누를 때마다 살아 있다
- 과하게 튀지 않고 세련됐다
- 오래 보고 있어도 피곤하지 않다

---

## UX 키워드

- 몰랑몰랑
- 부드러움
- 따뜻함
- 정갈함
- 고급스러움
- 저자극
- 촘촘한 마이크로인터랙션
- 촉감이 느껴지는 반응

---

## 기본 원칙

### 1. 모든 인터랙션은 살아 있어야 한다
버튼, 카드, 탭, 타일, 팝업, 배지, 수치 증가 모두 무반응이면 안 된다.

### 2. 모든 반응은 짧고 세련되어야 한다
길고 번잡한 애니메이션은 힐링을 망친다.

### 3. 모션은 정보 전달도 해야 한다
예쁜 것만이 아니라, 상태 변화가 읽혀야 한다.

### 4. 과도한 번쩍임보다 질감 있는 반응
광원, 반짝임, 파티클은 보조다.
핵심은 “움직임의 물성”이다.

---

## 화면 전환 방향

### 로비 → 퍼즐 진입
- 카드/보드가 살짝 확대되며 포커스 이동
- 화면 전체가 휙 바뀌기보다, 중심 요소가 연결되어야 함
- 180~320ms 범위 권장

### 퍼즐 종료 → 로비 복귀
- 결과 카드가 살짝 튀어나오고
- 자원이 로비 상단 자원바로 흘러들어가는 느낌
- “결과가 반영되었다”는 시각 연결 필요

### 팝업
- 아래에서 뜨거나 중앙에서 부드럽게 떠오름
- 배경은 살짝 블러/딤
- 닫힐 때도 급하게 사라지지 않음

---

## 버튼 인터랙션

### 눌림
- scale 0.97~0.99
- 그림자 살짝 감소
- 내부 하이라이트 변화

### 해제
- 짧은 복원
- 필요시 매우 약한 bounce

### 비활성
- 회색 처리만 하지 말고 질감도 줄인다
- 그러나 너무 죽지 않게

---

## 카드 인터랙션

### hover / focus
- 미세한 상승
- 밝기 또는 반사 변화
- 내부 아이콘/배지 미세 이동

### 선택
- border/글로우 강조
- 체크 또는 활성 표시
- 선택 순간 짧은 탭감

### 정렬/필터
- 카드가 재배열될 때 딱딱 끊기지 말고 자연스럽게 이동

---

## 2048 보드 모션

### 타일 생성
- 스케일 업 + 페이드 인
- 너무 튀지 않게

### 타일 이동
- 부드러운 ease-out
- 방향성이 읽혀야 함

### 타일 합체
- 겹침 → 압축 → 팡 → 상위 타일 등장
- 이때 가장 중요한 건 **압축감**이다.

### 상위 타일 등장
- 작은 하이라이트
- 숫자/아이콘 변화가 확실하게 보여야 함

### 보드 흔들림
- 필요 시 고단계 합체에서 아주 미세하게

---

## 수치 변화 표현

### 코인 증가
- 숫자가 단순 교체되지 말고 살짝 튀거나 롤링
- 획득 지점에서 HUD로 빨려 들어가는 느낌

### 경험치/진행도
- 바가 차오르는 속도와 타이밍을 조절
- 즉시 100% 점프보다 짧은 램프업 필요

### 주문 달성
- 체크 표시 / 배지 터짐 / 별 반짝임

---

## 로비 생동감 요소

항상 뭔가가 아주 약하게 움직여야 한다.

- 커피 김
- 컵 반사
- 전구/조명 잔떨림
- 메뉴판 작은 흔들림
- 식물 미세 흔들림
- 고객 이동
- 카운터 작은 작업 애니메이션

이런 요소가 많을수록 카페가 살아난다.

## 사운드 UX

사운드는 게임 감성의 절반이다.

### 버튼
- 짧고 둥근 클릭

### 카드 선택
- 조금 더 부드러운 톡

### 퍼즐 이동
- 거의 들릴락 말락한 미끄러짐

### 합체
- 묵직한 팝
- 상위 합체는 은은한 잔향

### 매장 판매
- 코인 소리, 컵 소리, 행복 반응음

### 배경
- 단색보다 그라데이션/은은한 텍스처
- 저채도 우드/크림

### 중경

### 전경

## 타격감과 힐링의 균형

이 게임은 액션 게임처럼 강한 타격감이 아니라,
**“부드럽지만 확실한 타격감”**이 맞다.

즉,
- 너무 세게 튀면 힐링이 깨진다.
- 너무 약하면 손맛이 없다.

적정선은
- 짧은 압축감
- 둥근 팝
- 작은 화면 반응
- 고급스러운 잔향

---

## 금지해야 할 UX 실수

- 모션이 길어서 답답함
- 중요한 반응과 사소한 반응의 강도가 비슷함
- 숫자만 바뀌고 물성이 없음
- 팝업이 갑자기 뜨고 사라짐
- 전체적으로 너무 번쩍거려 피곤함
- 모바일에서 입력 지연이 느껴짐

---

## 구현 우선순위

1. 입력 반응 속도
2. 타일 이동/합체 물성
3. 자원 획득 피드백
4. 로비 상호작용 반응
5. 화면 전환 통일감
6. 사운드/햅틱 연결
7. 생활감 연출

---

## 개발 시 체크리스트

아래 질문에 “예”가 많아야 한다.

- 버튼을 누를 때 살아 있는가?
- 타일이 합쳐질 때 기분 좋은가?
- 로비가 정지 화면처럼 보이지 않는가?
- 화면 전환이 어색하지 않은가?
- 수치 증가가 만족스러운가?
- 과장되지 않고 고급스러운가?
- 5분 만져도 피곤하지 않은가?

이 문서를 기준으로 UX 품질을 계속 검수한다.
