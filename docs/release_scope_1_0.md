# Release Scope 1.0

## 0. 문서 목적
이 문서는 현재 레포 기준으로 **1.0 출시선**을 고정하기 위한 문서다.

목표는 새 기능을 더 붙이는 것이 아니라,
**무엇을 1.0에 포함하고 무엇을 1.1 이후로 넘길지 분명하게 끊는 것**이다.

---

## 1. 1.0 제품 한 줄 정의
`Coffee 2048` 1.0은

**2048 퍼즐 -> 자원 획득 -> 카페 운영 -> 레벨/미션/해금 -> 저장 유지**

까지 닫히는 싱글플레이 루프를 제공하는 것을 목표로 한다.

## 4. 손님 메타의 1.0 판단
손님 메타는 **현재 구현 수준을 1.0 완료선**으로 둔다.

이유:
- 판매 write path와 이미 연결돼 있다
- 애정도 / 선호 보너스 / 스토리 / 단골 판정 / 날짜 경계가 저장과 함께 안정화돼 있다
- 로비와 카운터에 저자극 피드백으로 붙어 있어 코어 루프를 방해하지 않는다
- 반대로 주문/확장형 손님 시스템까지 키우면 1.0 범위를 넘는다

정리:
- **현재 손님 메타 v1은 1.0 포함**
- **확장형 손님 메타는 1.1 이후**

## 5. BM의 1.0 판단
BM은 1.0에서 **아주 좁게** 고정한다.

### 5-1. 포함
- 오프라인 보상 x2 보상형 광고
- 퍼즐 결과 x2 보상형 광고

### 5-2. 포함 조건
- 퍼즐 결과 x2는 **코인 + 원두만 2배**
- 성장 메타와 손님 메타의 진척은 배수 대상이 아니다
- 오프라인 x2도 판매 정산 배수 범위 안에서만 다룬다

### 5-3. 제외
- 미션 진행도 2배
- 손님 애정도/스토리/단골 판정 2배
- 도감/해금/시간대 메타 진척 2배
- 테마 스킨
- 이펙트 스킨
- pass/liveOps
- BM 실결제 확장

### 5-4. 현재 레포 기준 해석
- 현재 `/shop`은 placeholder 성격이며, 1.0 BM 확정 범위를 대표하지 않는다
- 1.0 BM 문구는 **출시선 결정**이고, 이번 작업에서 SDK 연동이나 실구현을 뜻하지 않는다

### 5-5. 1.0 비출시 표면 노출 정책
현재 레포에서는 1.0에 포함되지 않는 표면을 아래처럼 정리한다.

- `/shop`
  - **일반 진입점 숨김**
  - direct route에서는 **비출시 데모 보관함**으로만 남긴다
  - 실제 1.0 BM 화면처럼 읽히지 않게 한다
- `pass/liveOps`
  - **활성 기능처럼 노출하지 않음**
  - 화면이 남아 있더라도 **출시 후 예정 메모** 수준으로만 둔다
- 광고 제거 / 스타터 팩 / 테마 / 꾸미기 / 실결제 확장
  - **출시 후 예정**
  - 구매 버튼/진입점/문구를 1.0 기능처럼 보이지 않게 낮춘다
- 설비 업그레이드
  - 1.0 포함 기능이므로 유지한다
  - 단, `pass/liveOps`와 같은 확장 표면과 섞여 보이지 않게 분리한다

### 5-6. 1.0에 실제로 남는 BM 표면
1.0에서 실제로 남는 BM 표면은 아래뿐이다.

- 오프라인 보상 x2
- 퍼즐 결과 x2

추가 조건:

- 퍼즐 결과 x2는 **코인 + 원두만** 2배
- 미션 / 손님 / 도감 / 해금 / 시간대 메타 진척은 배수 대상이 아님
- 별도 상점 화면이나 pass/liveOps 화면을 1.0 BM 표면처럼 열어 두지 않음

## 9. 최종 원칙
1.0은 **넓히는 버전이 아니라 닫는 버전**이다.

지금부터는
- 주문/특별 원두/확장 손님 메타를 더 붙이기보다 미루고
- BM을 넓히기보다 `오프라인 x2`, `퍼즐 결과 x2(코인+원두만)`로 좁히고
- 손님 메타는 현재 v1 수준을 안정화하고
- 핵심 루프와 저장/회귀를 단단히 묶는 것이 우선이다.
